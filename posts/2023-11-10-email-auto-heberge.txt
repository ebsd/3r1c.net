email auto hébergé
2023-11-10

Je parviens désormais à recevoir mes mails seulement avec un script / service en
python : mail4one [1]. L'envoi se fait avec le service smtp2go [2] qui propose
en envoi gratuit pour 1000 mails par mois. Pour la réception mail4one [1] ne
prévoit pas de check spf et dkim, donc ceci peut engendrer du spam. Mais peut
être qu'un postgrey serait une solution palliative ? Notons que mail4one propose
aussi un serveur pop3.

[1]: https://gitlab.com/balki/mail4one
[2]: https://support.smtp2go.com/hc/en-gb/articles/223087947-Free-Plan

Client mail mutt
================

Configurer mutt pour pointer sur le dossiers /var/mails, et utiliser le relais
smtp de smtp2go.
```
$ sudo vi /etc/Muttrc
set folder="/var/mails"
set mask="!^\\.[^.]"
set mbox="/var/mails"
set record="+.Sent"
set postponed="+.Drafts"
set spoolfile="/var/mails"
set from="eric <me@3r1c.net>"
set smtp_url = "smtp://user:password@mail.smtp2go.com:587"
set ssl_starttls = yes
set edit_headers=yes
set markers=no
```
